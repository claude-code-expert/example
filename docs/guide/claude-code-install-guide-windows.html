Claude Code · Windows Setup Guide

윈도우에서 Claude Code 시작하기 — 설치부터 첫 실행까지

윈도우 사용자를 위한 Claude Code 설치 가이드입니다. 네이티브 설치와 WSL 설치 두 경로를 모두 다루고, Git 설치, 첫 로그인, 간단한 사용 예제, 자주 만나는 오류까지 순서대로 안내합니다. 터미널이 처음이어도 따라올 수 있습니다.

01두 가지 설치 경로 — 무엇을 고를까

윈도우에서 Claude Code를 쓰는 방법은 두 가지입니다. 네이티브 설치는 윈도우에 바로 설치하는 방식이고, WSL 설치는 윈도우 안에 리눅스(Ubuntu) 환경을 만들고 그 안에 설치하는 방식입니다.

경로필요한 것이런 분에게
A. 네이티브 윈도우없음 (Git for Windows는 선택·권장)윈도우 프로젝트·도구를 그대로 쓰고 싶다. 가장 빠르게 시작하고 싶다.
B. WSL2 강의 권장WSL2 활성화 (명령 한 줄 + 재부팅 1회)강의·책의 macOS/리눅스 명령어를 그대로 따라 하고 싶다. 샌드박싱(격리 실행)까지 쓰고 싶다.
이 강의에서는 WSL2를 권장합니다

강의 자료의 터미널 명령( ls, curl, chmod … )은 macOS/리눅스 기준입니다. WSL2 위에서는 이 명령들이 그대로 동작하므로 화면과 실습이 일치합니다. 다만 급하게 시작해야 한다면 경로 A(네이티브)도 충분합니다 — 둘 다 아래에 안내합니다.

시작 전 준비물

  • Windows 10 1809 이상 또는 Windows 11 · RAM 4GB 이상 (WSL2 병행 시 8GB 권장)
  • Claude 계정 — Pro 이상: 무료 플랜에는 Claude Code가 포함되지 않습니다.
  • 안정적인 인터넷 연결

02경로 A — 네이티브 윈도우 설치

2-1. PowerShell 열기

시작 메뉴에서 PowerShell을 검색해 실행합니다. 관리자 권한은 필요 없습니다. 프롬프트가 PS C:\Users\사용자> 처럼 PS로 시작하면 PowerShell이 맞습니다.

2-2. 설치 명령 실행

irm https://claude.ai/install.ps1 | iex

설치가 끝나면 설치 위치(Location)와 버전 정보가 출력됩니다.

CMD(명령 프롬프트)를 쓰고 있다면

프롬프트가 C:\Users\사용자> 처럼 PS 없이 시작하면 CMD입니다. CMD에서는 위 명령 대신 아래를 사용합니다.

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

2-3. PATH 환경 변수 등록

설치 후 claude 명령이 인식되지 않으면, 그림 2의 Location에 나온 경로 (C:\Users\[사용자]\.local\bin)를 PATH에 추가해야 합니다.

  1. Win + R → sysdm.cpl 입력 → 시스템 속성 창이 열립니다.
  1. 고급 탭 → [환경 변수] 버튼 클릭
  1. 사용자 변수 목록에서 Path 선택 → [편집]
  1. [새로 만들기] → 복사해 둔 C:\Users\[사용자]\.local\bin 붙여넣기 → [확인]

2-4. 설치 확인

기존 창은 닫고 새 PowerShell을 연 뒤 버전을 확인합니다. 환경 변수는 새로 연 터미널부터 적용됩니다.

claude --version

03Git for Windows 설치 네이티브 경로 권장

네이티브 윈도우에서 Git for Windows를 설치하면 Claude Code가 Git Bash를 통해 Bash 도구를 쓸 수 있게 됩니다 (없으면 PowerShell로 대체 동작). 버전 관리를 위해서도 사실상 필수입니다. WSL 경로(B)를 쓴다면 이 단계는 건너뜁니다 — Ubuntu에 git이 기본 포함됩니다.

3-1. 다운로드

https://git-scm.com/downloads 에서 Windows용 Standalone Installer(64-bit)를 내려받아 실행합니다. (Portable은 무설치 압축판 — 일반적으로는 Standalone을 선택합니다.)

한 줄로 끝내고 싶다면 (winget)
winget install --id Git.Git -e --source winget

설치 마법사 없이 기본 옵션으로 설치됩니다. 아래 3-2는 설치 파일을 직접 실행할 때의 안내입니다.

3-2. 설치 마법사 — 대부분 기본값, 멈춰서 볼 곳은 세 군데

설치 화면이 10개 이상 나오지만 대부분 기본값 그대로 Next를 누르면 됩니다. 의미를 알고 지나가면 좋은 화면 세 곳만 짚습니다.

화면초보자 권장 선택
① 기본 에디터 선택
(Choosing the default editor)		기본값은 Vim인데 터미널 에디터가 낯설면 부담됩니다. 목록에서 Visual Studio Code(설치돼 있다면)나 Notepad++를 선택하세요.
② PATH 환경 설정
(Adjusting your PATH)		가운데 옵션 "Git from the command line and also from 3rd-party software"(기본값) — PowerShell·CMD·다른 프로그램에서 git 명령을 쓸 수 있게 합니다.
③ 줄바꿈 처리
(Configuring the line ending conversions)		첫 번째 "Checkout Windows-style, commit Unix-style line endings"(기본값) — 체크아웃 시 CRLF, 커밋 시 LF로 자동 변환해 macOS/리눅스 협업자와 충돌을 막습니다.

나머지 화면은 기본값이 안전합니다 — 요약하면:

  • Components: 기본 체크 그대로 (Git Bash Here, Git LFS 포함)
  • 기본 브랜치 이름: "Let Git decide" 그대로
  • SSH / HTTPS: bundled OpenSSH · OpenSSL library 그대로
  • 터미널: MinTTY(기본) — 리눅스와 유사한 터미널 환경
  • git pull 기본 동작: Default (fast-forward or merge)
  • 자격 증명: Git Credential Manager — GitHub 인증 정보를 한 번 입력하면 이후 자동 사용
  • Extra options: Enable file system caching 체크 그대로

3-3. 확인

git --version
# git version 2.x.x.windows.x 형태가 나오면 성공

탐색기에서 아무 폴더나 우클릭 → Git Bash Here가 보이면 그 폴더에서 바로 Git Bash 터미널을 열 수 있습니다.

Claude Code가 Git Bash를 못 찾는다면

설치 경로가 다르면 settings.json에 경로를 지정합니다.

{
  "env": {
    "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe"
  }
}

04경로 B — WSL2 설치

WSL(Windows Subsystem for Linux)은 윈도우 안에서 진짜 리눅스를 실행하는 마이크로소프트 공식 기능입니다. 명령 한 줄로 Ubuntu가 설치됩니다.

4-1. PowerShell을 관리자로 실행

시작 메뉴에서 PowerShell 검색 → 우클릭 → 관리자 권한으로 실행. (WSL 설치에만 관리자가 필요합니다.)

4-2. 설치 명령 한 줄

wsl --install

WSL2와 Ubuntu(기본 배포판)가 함께 설치됩니다. 설치 중 재부팅이 한 번 필요하고, Ubuntu 이미지 다운로드에 네트워크 속도에 따라 몇 분 걸릴 수 있습니다 — 걸어두고 기다리면 됩니다.

4-3. Ubuntu 첫 실행 — 계정 만들기

재부팅 후 Ubuntu 터미널이 자동으로 열리며(안 열리면 시작 메뉴에서 Ubuntu 실행) 리눅스용 사용자 이름과 비밀번호를 새로 만듭니다. 윈도우 계정과 별개이며, 비밀번호 입력 시 화면에 아무것도 표시되지 않는 것이 정상입니다.

4-4. WSL2인지 확인

PowerShell(일반 권한)에서:

wsl --list --verbose
#   NAME      STATE           VERSION
# * Ubuntu    Running         2        ← VERSION이 2면 정상
VERSION이 1로 나온다면

wsl --set-version Ubuntu 2 로 올려줍니다. WSL2에서만 Claude Code의 샌드박싱(명령 격리 실행)이 지원됩니다.

05WSL 안에 Claude Code 설치

여기부터는 Ubuntu 터미널 안에서 진행합니다. PowerShell이나 CMD가 아닙니다 — 설치도 실행도 모두 WSL 터미널에서 합니다.

5-1. 설치 (리눅스용 명령 그대로)

curl -fsSL https://claude.ai/install.sh | bash

5-2. 확인

claude --version
claude doctor   # 설치·PATH·업데이트 상태 한 번에 점검
command not found가 나온다면

터미널을 닫았다 다시 열어보고, 그래도 안 되면 ~/.bashrc 맨 아래에 다음 한 줄을 추가한 뒤 source ~/.bashrc:

export PATH="$HOME/.local/bin:$PATH"
프로젝트는 리눅스 파일시스템에 두세요

WSL에서 /mnt/c/…(윈도우 드라이브) 아래 파일을 다루면 리눅스 파일시스템 대비 수십 배 느린 I/O가 발생합니다. 작업 폴더는 홈 디렉터리(~/projects/…) 아래에 만드는 것을 권장합니다.

06첫 실행 — 로그인과 사용 예제

네이티브·WSL 어느 경로든 이후 사용법은 같습니다. (네이티브는 PowerShell에서, WSL은 Ubuntu 터미널에서.)

6-1. 연습용 폴더 만들고 실행

# WSL(Ubuntu) 기준
mkdir -p ~/projects/hello-claude && cd ~/projects/hello-claude
claude
# 네이티브 윈도우(PowerShell) 기준
mkdir C:\projects\hello-claude; cd C:\projects\hello-claude
claude

첫 실행 시 브라우저가 열리며 로그인을 요청합니다. Claude 계정(Pro 이상)으로 로그인하면 터미널로 돌아와 세션이 시작됩니다.

6-2. 첫 대화 예제

프롬프트에 자연어로 입력합니다. 예를 들어:

> 인사말을 출력하는 hello.py 파일을 만들어줘. 실행 방법도 알려줘.

Claude가 파일을 만들겠다고 제안하면 승인(Enter)합니다. 만들어진 파일은 같은 폴더에서 바로 확인할 수 있습니다.

> 이 폴더 구조를 설명해줘

기존 프로젝트 폴더에서 실행했다면 /init 을 입력해 보세요 — 코드베이스를 분석해 프로젝트 안내 파일(CLAUDE.md)을 자동으로 만들어 줍니다.

6-3. 자주 쓰는 기본 명령

입력동작
claude대화형 세션 시작 (프로젝트 폴더에서 실행)
/init코드베이스 분석 → CLAUDE.md 생성
/help사용 가능한 명령 목록
/exit 또는 Ctrl+C 두 번세션 종료
claude doctor설치 상태 진단 (터미널에서)

07API 키·환경 변수는 어디에 저장하나

MCP 서버 설정(.mcp.json)의 ${GITHUB_PAT} 같은 자리표시자는 claude를 실행한 터미널의 환경 변수에서 값을 가져옵니다. 저장 위치가 경로에 따라 다릅니다.

환경저장 위치명령
네이티브 윈도우사용자 환경 변수 (레지스트리) — 새 터미널부터 적용setx GITHUB_PAT "ghp_xxxx"
WSL(Ubuntu)WSL 안의 ~/.bashrc — 윈도우 환경 변수와 별개(서로 공유 안 됨)export GITHUB_PAT="ghp_xxxx"
흔한 실수 두 가지

① setx로 저장한 변수는 지금 열려 있는 창에는 적용되지 않습니다 — 새 터미널을 여세요. ② WSL에서 쓸 키를 윈도우 환경 변수에 넣으면 WSL에서는 보이지 않습니다 — WSL용 키는 WSL 안 ~/.bashrc에.

08문제 해결

증상원인과 해결
'irm'은(는) … 인식되지 않습니다지금 CMD에 있습니다. PowerShell을 열거나(프롬프트가 PS로 시작), CMD용 설치 명령(02 참고)을 쓰세요.
The token '&&' is not a valid statement separator지금 PowerShell에 있습니다. CMD용 명령을 PowerShell에 붙여넣은 경우 — PowerShell용 irm … | iex를 쓰세요.
claude : … 인식되지 않습니다 / command not foundPATH 미등록이거나 옛 터미널. ① 새 터미널 열기 → ② 네이티브는 02-3의 PATH 등록, WSL은 05-2의 ~/.bashrc PATH 추가.
WSL 설치 후 Ubuntu가 안 열림재부팅했는지 확인 → 시작 메뉴에서 Ubuntu 직접 실행 → 그래도 안 되면 wsl --update 후 재시도.
WSL에서 파일 작업이 너무 느림프로젝트가 /mnt/c/… 아래에 있는 경우입니다. ~/projects/…로 옮기세요 (05 참고).
로그인 브라우저가 안 열림 (WSL)터미널에 출력된 URL을 복사해 윈도우 브라우저에 직접 붙여넣으면 됩니다.

원인이 안 잡히면 진단 명령부터:

claude doctor   # 설치 방식·PATH·업데이트 상태 한 번에 점검

그래도 해결이 안 되면 실행한 설치 명령과 전체 에러 출력을 모아 GitHub Issues에서 동일 사례를 검색하세요.
