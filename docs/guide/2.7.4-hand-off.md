## 2.7.4 PreCompact Hook으로 작업 흐름 이어가기

개발을 진행하다 보면 긴 세션 동안 작업하다 컨텍스트 윈도우가 가득 차서 `/compact` 명령이 실행된다. 이때 중요한 작업 내역이 요약 과정에서 손실될 수 있다. 또 IDE 크래시나 네트워크 문제로 세션 중단 시에도 문제가 될 수 있다.

이런 경우 **PreCompact Hook**을 활용하면 compact 직전에 자동으로 handoff 문서를 생성하여 작업 컨텍스트를 보존할 수 있다. 주로 세션 간 작업 인계를 위해서 `HANDOFF.md`라는 구조화된 문서를 작성해서 이전 작업을 자동으로 참조하게 하거나, Hook 설정으로 compact 시점에 자동 생성되도록 구성해두면 유용하다.

### PreCompact Hook의 matcher 이해

PreCompact Hook의 `matcher`는 compact가 어떻게 트리거됐는지를 구분한다. 이 설정이 "완전 자동"과 "반자동"을 가르는 핵심이다.

[표 2-22] PreCompact matcher 종류별 동작

| matcher 값 | 트리거 조건 | 설명 |
|-----------|-----------|------|
| `"auto"` | 컨텍스트 윈도우 ~95% 도달 시 자동 compact | 사용자 개입 없이 발동 |
| `"manual"` | 사용자가 `/compact` 직접 실행 | 의도적 compact만 대응 |
| `""` 또는 `"*"` | auto + manual 모두 | **완전 자동에 필요한 설정** |

> 📌 `matcher`를 빈 문자열 `""`이나 `"*"`로 설정하면 auto-compact에서도 수동 `/compact`에서도 모두 Hook이 실행된다. 이것이 완전 자동 handoff의 핵심이다.

- 출처: [Claude Code Hooks reference](https://code.claude.com/docs/en/hooks) · [Claude 공식 블로그 — How to configure hooks](https://claude.com/blog/how-to-configure-hooks)

### 방법 1: 플러그인으로 완전 자동 구성 (who96/claude-code-context-handoff)

가장 간편한 방법은 전용 플러그인을 설치하는 것이다. `who96/claude-code-context-handoff`는 auto-compact, 수동 `/compact`, `/clear` 모든 상황에서 **사용자 개입 없이** 자동으로 컨텍스트를 보존하고 복원한다.

```bash
# 설치
git clone https://github.com/who96/claude-code-context-handoff.git
cd claude-code-context-handoff
./install.sh
```

설치 스크립트가 자동으로 수행하는 작업:
- `~/.claude/hooks/`에 스크립트 복사
- `~/.claude/settings.json`에 PreCompact, SessionEnd, SessionStart Hook 등록 (기존 설정 백업)
- `~/.claude/handoff/` 디렉토리 생성

**동작 흐름:**

```
auto-compact 또는 /compact 발생
    → PreCompact Hook이 자동으로 handoff 작성
    → compact 완료 후 SessionStart(compact)에서 additionalContext로 복원

/clear 실행
    → SessionEnd(clear) Hook이 handoff 작성
    → SessionStart(clear)에서 handoff 복원
```

**핵심 특징:**
- PreCompact Hook이 `auto`와 `manual` 모두 매칭되어 어떤 compact든 자동 실행
- 별도 API 호출 없이 트랜스크립트에서 직접 컨텍스트를 추출 (마지막 15개 사용자 메시지, 10개 어시스턴트 응답, 파일 경로 등)
- `~/.claude/handoff/<session_id>.md`에 자동 저장
- SessionStart Hook이 `additionalContext`로 자동 주입하여 새 세션에서 즉시 이어서 작업 가능
- 선택적 Supervisor 모드: `/compact`을 `/clear`로 자동 치환하여 더 깔끔한 세션 전환

**환경 변수로 커스터마이징 가능:**

```bash
export HANDOFF_MAX_USER_MESSAGES=20      # 보존할 사용자 메시지 수 (기본: 15)
export HANDOFF_MAX_ASSISTANT_CHARS=1000   # 어시스턴트 응답 최대 문자수
export HANDOFF_DEDUP_THRESHOLD=0.90       # 중복 제거 임계값
export HANDOFF_LATEST_MAX_AGE_SEC=900     # 최신 handoff 유효 시간 (초)
```

- 출처: [who96/claude-code-context-handoff (GitHub)](https://github.com/who96/claude-code-context-handoff)

### 방법 2: settings.json에 직접 Hook 작성 (DIY)

플러그인 설치 없이 `settings.json`에 직접 Hook을 등록할 수도 있다. 가장 가벼운 방법이면서 동작 원리를 이해하기 좋다.

```json
{
  "hooks": {
    "PreCompact": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "python3 ~/.claude/hooks/pre-compact-handoff.py"
          }
        ]
      }
    ],
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "cat HANDOFF.md 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

SessionStart Hook의 stdout은 자동으로 Claude의 컨텍스트에 추가된다. 따라서 `cat HANDOFF.md`의 출력이 새 세션의 초기 컨텍스트가 된다.

**Python 스크립트 예시 (`~/.claude/hooks/pre-compact-handoff.py`):**

```python
#!/usr/bin/env python3
"""
PreCompact Hook: 트랜스크립트를 읽어 HANDOFF.md를 자동 생성
"""
import json
import sys
from pathlib import Path
from datetime import datetime

# stdin으로 세션 정보 수신
input_data = json.load(sys.stdin)
session_id = input_data.get("session_id", "unknown")
transcript_path = input_data.get("transcript_path", "")
cwd = input_data.get("cwd", ".")
trigger = input_data.get("trigger", "unknown")

# 트랜스크립트 파싱
messages = []
if transcript_path and Path(transcript_path).exists():
    with open(transcript_path, "r") as f:
        for line in f:
            try:
                entry = json.loads(line.strip())
                if entry.get("type") in ("human", "assistant"):
                    messages.append(entry)
            except json.JSONDecodeError:
                continue

# 최근 메시지 추출 (마지막 20개)
recent = messages[-20:] if len(messages) > 20 else messages
user_msgs = [m for m in recent if m.get("type") == "human"]
assistant_msgs = [m for m in recent if m.get("type") == "assistant"]

# HANDOFF.md 생성
handoff_path = Path(cwd) / "HANDOFF.md"
with open(handoff_path, "w") as f:
    f.write(f"# Session Handoff\n\n")
    f.write(f"**생성일**: {datetime.now().strftime('%Y-%m-%d %H:%M')}\n")
    f.write(f"**세션 ID**: {session_id}\n")
    f.write(f"**트리거**: {trigger}\n\n")
    f.write(f"## 최근 작업 컨텍스트\n\n")
    for msg in user_msgs[-5:]:
        content = str(msg.get("message", {}).get("content", ""))[:200]
        f.write(f"- {content}\n")
    f.write(f"\n## 어시스턴트 마지막 응답 요약\n\n")
    for msg in assistant_msgs[-3:]:
        content = str(msg.get("message", {}).get("content", ""))[:300]
        f.write(f"- {content}\n")

sys.exit(0)
```

> ⚠️ 트랜스크립트 JSONL의 내부 구조는 Claude Code 버전에 따라 다를 수 있으므로, 프로젝트에 맞게 파싱 로직을 조정해야 한다.

### 방법 3: Haiku 인스턴스로 AI 요약 생성

트랜스크립트를 별도 Haiku 인스턴스에 보내 AI가 요약하는 고급 방법이다. 요약 품질이 가장 높지만 API 비용이 발생하고 약 20초의 추가 시간이 소요된다.

```json
{
  "hooks": {
    "PreCompact": [
      {
        "matcher": "auto",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/pre-compact-handover.py"
          }
        ]
      }
    ]
  }
}
```

이 방식의 핵심은 `claude -p` 또는 `claude --resume <session> --fork-session --model haiku --print`로 별도 Claude 인스턴스를 생성하여 요약을 위임하는 것이다. 현재 세션의 컨텍스트 윈도우가 이미 가득 찬 상태이기 때문에, 현재 세션 내에서 요약을 시도하면 공간이 부족하다. 별도 인스턴스를 사용하면 이 문제를 우회할 수 있다.

> 📌 `matcher`를 `"auto"`로 설정하면 auto-compact에서만 실행되고, 수동 `/compact`에서는 실행되지 않는다. 사용자가 의도적으로 `/compact`을 실행하는 경우에는 별도 handoff가 불필요하다는 판단이다.

- 출처: [Auto Memory & PreCompact Hooks Explained (Yuanchang's Blog)](https://yuanchang.org/en/posts/claude-code-auto-memory-and-hooks/) · [Mother CLAUDE: Automating Everything with Hooks (DEV Community)](https://dev.to/dorothyjb/mother-claude-automating-everything-with-hooks-12jh)

### 참고: 수동 Handoff 플러그인

완전 자동이 아닌 **수동 실행** 방식의 Handoff 플러그인도 존재한다. 상황에 따라 직접 handoff를 생성하고 싶을 때 유용하다.

**willseltzer/claude-handoff** — 슬래시 커맨드 방식:

```bash
/plugin marketplace add willseltzer/claude-handoff
/plugin install handoff
```

설치 후 `/handoff:create` (전체 handoff), `/handoff:quick` (간략 handoff), `/handoff:resume` (이전 handoff로 재개)를 수동으로 실행한다. PreCompact Hook이 등록되는 것이 아니므로 `/compact` 전에 직접 실행해야 한다.

**kylesnowschwartz/claude-handoff** — `/compact handoff:<목표>` 방식:

```bash
claude plugin marketplace add kylesnowschwartz/claude-handoff
claude plugin install claude-handoff
```

일반 `/compact`은 평소대로 동작하고, `/compact handoff:칸반 보드 구현 계속`처럼 `handoff:` 접두어를 붙여야만 PreCompact Hook이 활성화된다. 목표 지향적 handoff가 필요할 때 적합하다.

- 출처: [willseltzer/claude-handoff (GitHub)](https://github.com/willseltzer/claude-handoff) · [kylesnowschwartz/claude-handoff (GitHub)](https://github.com/kylesnowschwartz/claude-handoff)

### Handoff 문서 작성 가이드라인

Handoff 문서는 너무 길 경우 토큰이 낭비될 수 있기 때문에 다음 원칙을 지켜야 한다.

- **최근 문서 위주로 관리**: 오래된 handoff는 정리하거나 아카이빙한다
- **에러 메시지와 코드 컨텍스트를 명확하게 포함**: "에러 발생"이 아닌 실제 에러 메시지를 기록한다
- **실패한 접근법은 필수 기재**: 다음 세션에서 같은 시행착오를 반복하지 않도록 한다. willseltzer/claude-handoff의 README에서도 이를 handoff에서 가장 가치 있는 부분이라고 강조한다
- **Auto Memory와 병행 사용**: 클로드 코드의 Auto Memory(`~/.claude/projects/`)는 장기 지식을, PreCompact Hook의 Handoff는 작업 상태 스냅샷을 담당한다. 두 메커니즘은 경쟁이 아니라 보완 관계다
