이 프로젝트의 보안 상태를 점검해줘. 아래 체크리스트를 기준으로 현재 코드베이스를 분석하고, 각 항목의 통과/미통과/해당없음 여부를 판정해줘.

## 1. 입력 검증
- [ ] 서버 사이드에서 모든 사용자 입력을 검증하는가 (Zod 등 런타임 검증)
- [ ] 문자열 길이 제한이 적용되어 있는가
- [ ] 타입 검증이 런타임에서도 수행되는가

## 2. SQL Injection 방지
- [ ] ORM(Prisma 등)을 통한 쿼리만 사용하는가
- [ ] Raw SQL 쿼리가 존재하지 않는가
- [ ] 사용자 입력이 쿼리에 직접 삽입되는 곳이 없는가

## 3. XSS 방지
- [ ] dangerouslySetInnerHTML 사용이 없는가
- [ ] 사용자 입력을 HTML에 직접 렌더링하지 않는가
- [ ] React 자동 이스케이핑에 의존하고 있는가

## 4. 환경변수 & 시크릿
- [ ] .env 파일이 .gitignore에 포함되어 있는가
- [ ] 하드코딩된 API 키나 비밀번호가 없는가
- [ ] DATABASE_URL 등 민감한 값이 환경변수로 관리되는가

## 5. 의존성 보안
- [ ] npm audit 결과 critical/high 취약점이 없는가
- [ ] 불필요한 의존성이 없는가

## 6. API 보안
- [ ] 에러 응답에서 스택 트레이스나 내부 정보가 노출되지 않는가
- [ ] 적절한 HTTP 상태 코드를 반환하는가
- [ ] Rate limiting 또는 요청 크기 제한이 있는가

## 7. 권한 설정 (Claude Code)
- [ ] settings.json에 위험 명령어 deny 규칙이 있는가 (rm -rf, sudo, git push --force 등)
- [ ] --dangerously-skip-permissions를 프로덕션에서 사용하지 않는가

결과를 표 형태로 정리하고, 발견된 문제에 대해 수정 방법을 제안해줘.
