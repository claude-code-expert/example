01언인스톨 (Uninstall)

1-1. 설치 방식부터 확인

설치 방식마다 파일 위치가 달라, 잘못된 언인스톨 명령이 "반쯤 지워진 설치"의 가장 흔한 원인입니다. 먼저 바이너리 경로를 확인합니다.

which claude
which -a claude   # PATH 상의 모든 claude 나열 (중복 설치 확인)
스크린샷 영역which -a claude 결과 — /Users/<사용자>/.local/bin/claude 한 줄만 나오는 정상 케이스

이번 시나리오는 curl ... install.sh | bash로 설치했으므로 native installation입니다. 출력이 ~/.local/bin/claude이면 아래로 진행합니다. 여러 줄이면 중복 설치 → 8번 트러블슈팅 참조.

1-2. native 언인스톨 (이번 케이스)

rm -f ~/.local/bin/claude
rm -rf ~/.local/share/claude
  • ~/.local/bin/claude : 실행 바이너리
  • ~/.local/share/claude : 버전 관리 파일

1-3. 설정 파일 처리 (선택 · 의사결정 포인트)

주의

아래를 지우면 설정·허용 도구·MCP 서버 구성·세션 히스토리가 모두 삭제됩니다. 재설치 후 그대로 쓸 거면 백업하거나 남겨두세요.

# (선택) 백업 — 재설치 후 복원용
cp -R ~/.claude ~/.claude.backup
cp ~/.claude.json ~/.claude.json.backup

# (선택) 완전 초기화 — 깨끗한 첫 화면을 보여주고 싶을 때만
rm -rf ~/.claude
rm ~/.claude.json
팁

"깨끗한 첫 설치 화면"을 녹화하려면 설정도 삭제, "기존 세팅 유지 재설치"를 보여주려면 남겨두세요. VS Code 확장·JetBrains 플러그인·Desktop 앱이 깔려 있으면 실행 시 ~/.claude/를 다시 생성합니다.

1-4. 언인스톨 검증

claude --version
  • 정상이면 zsh: command not found: claude 출력
  • 여전히 버전이 나오면 → 중복 설치 또는 오래된 셸 alias 잔존 신호 (8번)
스크린샷 영역언인스톨 후 command not found 에러 화면

02재설치 (Install)

2-1. native 설치 (권장)

curl -fsSL https://claude.ai/install.sh | bash
참고

native 설치는 백그라운드 자동 업데이트를 지원합니다. (Homebrew·WinGet·npm은 수동 업데이트)

설치 완료 시 보통 다음 메시지가 출력됩니다.

✔ Claude Code successfully installed!
  Version: <버전>
  Location: ~/.local/bin/claude
스크린샷 영역설치 완료 메시지 + (PATH 미설정 시) ~/.local/bin is not in your PATH 경고
주의

설치 직후 ~/.local/bin is not in your PATH 경고가 뜨면 4번(PATH 설정)으로 바로 이동하세요.

2-2. 특정 버전 / stable 채널 (버전 고정이 필요할 때)

# 최신 버전 (기본)
curl -fsSL https://claude.ai/install.sh | bash

# stable 채널 (약 1주 지연, 큰 회귀 버그 스킵)
curl -fsSL https://claude.ai/install.sh | bash -s stable

# 특정 버전 고정 (예시 버전)
curl -fsSL https://claude.ai/install.sh | bash -s 2.1.89
팁

화면과 버전을 일치시키려면 특정 버전 고정이 유용합니다. 단, 자동 업데이트로 올라갈 수 있으니 필요 시 settings.json에 채널/최소버전을 명시하세요.

2-3. 대안 설치 방법 (참고)

# Homebrew — PATH 자동 처리, 자동 업데이트 X (brew upgrade 필요)
brew install --cask claude-code

# npm 글로벌 — Node.js 18+ 필요, sudo 금지
npm install -g @anthropic-ai/claude-code
주의

sudo npm install -g는 권한·보안 문제를 유발하므로 사용 금지입니다. 권한 에러 시 native installer로 전환하세요.

04PATH 설정 (zshrc / bashrc)

native 설치는 바이너리를 ~/.local/bin/claude에 둡니다. 셸이 이 디렉터리를 모르면 command not found가 납니다.

4-1. PATH에 이미 들어있는지 확인

echo $PATH | tr ':' '\n' | grep -Fx "$HOME/.local/bin"
  • /Users/<사용자>/.local/bin 출력 → 이미 설정됨 (6번 실행으로)
  • 아무것도 안 나옴 → 아래에서 추가

4-2. zsh (macOS 기본 셸) — 영구 설정

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

4-3. bash 사용 시 — 영구 설정

리눅스/WSL 등 대부분은 ~/.bashrc로 충분합니다.

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
주의 (macOS bash)

macOS에서 bash는 로그인 셸일 때 ~/.bashrc가 아니라 ~/.bash_profile(또는 ~/.profile)을 읽습니다. 새 터미널 창에서 적용이 안 되면 ~/.bash_profile이 ~/.bashrc를 불러오도록 한 번 연결해 두세요.

# .bash_profile 에서 .bashrc 를 불러오게 연결 (최초 1회)
echo '[ -f ~/.bashrc ] && . ~/.bashrc' >> ~/.bash_profile
source ~/.bash_profile

4-4. 셸별 설정 파일 정리

어느 파일에 넣어야 하는지 헷갈릴 때 참고하세요.

셸PATH를 넣을 파일비고
zsh
(macOS 기본)		~/.zshrc대부분 이것으로 충분. 로그인 셸에서만 빠지면 ~/.zprofile에도 동일 추가
bash
(macOS)		~/.bashrc + ~/.bash_profile 연결로그인 셸이 .bashrc를 안 읽으므로 위 4-3 연결 필요
bash
(Linux/WSL)		~/.bashrc연결 없이 보통 바로 동작
fish / nushell각 셸 전용 설정각 셸 문법으로 ~/.local/bin을 PATH에 추가
팁

본인 셸 확인: echo $SHELL → /bin/zsh면 zsh, /bin/bash면 bash. macOS는 2019년부터 기본이 zsh이므로 대부분 ~/.zshrc만 수정하면 됩니다. 두 셸을 오가면 양쪽 모두 추가하세요.

주의

큰따옴표 안의 ~는 홈 경로로 확장되지 않습니다. 반드시 "$HOME/.local/bin:$PATH" 형태로 작성하세요. (자세한 함정은 8번 T-2)

4-5. 적용 확인

claude --version
스크린샷 영역PATH 추가 → source → claude --version 정상 출력까지의 연속 화면

05버전 확인이 안 되는 케이스 — 원인과 해결

강의 핵심 파트. 수강생이 가장 많이 막히는 지점입니다.

5-1. command not found: claude — PATH 누락 (가장 흔함)

원인: 설치는 됐지만 ~/.local/bin이 PATH에 없거나, PATH는 추가했는데 현재 셸이 아직 못 읽음.

# 1) 새 터미널을 열거나
source ~/.zshrc

# 2) 바이너리가 실제로 있는지 확인
ls -la ~/.local/bin/claude   # 없으면 설치 실패 → 2번 재설치
스크린샷 영역ls -la ~/.local/bin/claude로 바이너리 존재 확인

5-2. 설치 스크립트가 HTML/403 반환 (syntax error near unexpected token '<')

원인: 설치 URL이 스크립트 대신 HTML/에러 반환. 네트워크·지역 라우팅·일시 장애, 또는 미지원 국가.

# 다운로드 서버 도달 확인 (HTTP/2 200 이면 도달 성공 → 재시도)
curl -sI https://downloads.claude.ai/claude-code-releases/latest

# 대안: Homebrew로 설치
brew install --cask claude-code
참고

"App unavailable in region"이 보이면 지원 국가가 아닙니다.

5-3. 버전이 이상하다 / 언인스톨했는데 아직 실행됨 — 중복 설치

원인: native + npm 글로벌 + 레거시 로컬 npm 공존. 어디서 왔는지 모두 확인 후 하나만 남깁니다.

which -a claude                                   # 모든 claude
ls -la ~/.local/bin/claude                        # native installer
ls -la ~/.claude/local/                           # 레거시 로컬 npm
npm -g ls @anthropic-ai/claude-code 2>/dev/null   # npm 글로벌

# 정리 (native만 남기기 권장)
npm uninstall -g @anthropic-ai/claude-code
rm -rf ~/.claude/local

5-4. macOS에서 dyld: cannot load / Symbol not found / Abort trap: 6

원인: macOS 버전이 바이너리 요구사항보다 낮음. Claude Code는 macOS 13.0+ 필요.

해결: 사과 메뉴 → "이 Mac에 관하여"에서 버전 확인 후 macOS 업데이트. (Homebrew도 같은 바이너리라 해결 안 됨)

5-5. Illegal instruction

원인: CPU가 요구 명령어셋(AVX 등) 미지원. 주로 2013년 이전 CPU나 일부 VM. 해결: native 바이너리 우회책 없음 (Apple Silicon Mac에서는 거의 발생하지 않음).

5-6. 로그인 후 403 Forbidden (설치는 됐으나 인증 실패)

원인: 구독 비활성, 또는 셸에 남은 ANTHROPIC_API_KEY가 구독 OAuth를 덮어씀.

unset ANTHROPIC_API_KEY
claude
# ~/.zshrc, ~/.bashrc, ~/.profile 에서 export ANTHROPIC_API_KEY=... 줄 삭제
팁

세션 내 /status로 현재 활성 인증 방식을 확인할 수 있습니다.

06Claude 실행 (Run)

주의

반드시 작업할 프로젝트 폴더에서 실행하세요. Desktop이나 홈 디렉터리에서 실행하면 안 됩니다.

cd ~/projects/my-app   # 프로젝트 루트로 이동
claude                 # 대화형 세션 시작

최초 실행 시 브라우저가 열리며 로그인합니다(Pro/Max/Team/Enterprise/Console 계정 필요). 무료 Claude.ai 플랜은 Claude Code 미포함.

스크린샷 영역claude 최초 실행 — 브라우저 OAuth 로그인 흐름
흐름도세션 시작 흐름: cd 프로젝트 → claude 실행 → (최초) 브라우저 로그인 → REPL 진입 → 프롬프트 입력 대기

07실행 시 기본 옵션 (macOS)

참고

claude --help가 모든 플래그를 보여주진 않습니다. 아래는 공식 CLI reference 기준 실무 빈출 옵션입니다.

7-1. 기본 실행 형태

명령설명
claude대화형 세션(REPL) 시작
claude "질문"초기 프롬프트와 함께 세션 시작
claude -p "질문"1회성 질의 후 종료 (print 모드, 스크립트/CI용)
cat file | claude -p "분석"파이프 입력 처리
claude update최신 버전 수동 업데이트
claude doctor설치/설정 진단
claude auth login로그인 (--console로 Console 과금 로그인)

7-2. 세션 관리

플래그설명
-c, --continue현재 디렉터리의 가장 최근 대화 이어가기
-r, --resume <id|name>특정 세션 재개 (인자 없으면 선택 UI)
-n, --name "이름"세션 표시 이름 지정
--fork-session재개 시 원본 보존, 새 세션 ID로 분기
--session-id <UUID>특정 세션 ID 지정
 
claude -c                          # 직전 대화 이어가기
claude --resume auth-refactor      # 이름으로 세션 재개

7-3. 모델 / 추론 강도

플래그설명
--model모델 지정 (sonnet·opus 별칭 또는 풀네임)
--effort추론 강도 low/medium/high/xhigh/max (모델별 가용 범위 상이)
 
claude --model opus
claude --effort high
참고

모델 별칭/풀네임은 시점에 따라 달라질 수 있습니다. 녹화 시 claude --help 또는 공식 문서로 현재 값을 확인하세요.

7-4. 권한 모드 (안전 ↔ 자동화 트레이드오프)

플래그설명
--permission-mode <mode>시작 권한 모드 지정

허용값: default, acceptEdits, plan, auto, dontAsk, bypassPermissions

claude --permission-mode plan
claude --allowedTools "Read" "Bash(git log *)"
주의

--dangerously-skip-permissions는 샌드박스 환경에서만 신중히 사용하세요. 세션 중에는 Shift+Tab으로 권한 모드를 순환 전환할 수 있습니다.

08트러블슈팅 — 자동 PATH 등록이 안 될 때

설치 스크립트가 PATH를 자동 등록해도 셸 환경에 따라 claude를 못 찾는 경우가 잦습니다. 아래는 zsh/bash에서 가장 흔한 원인 순입니다.

T-1. source 안 함 / 새 터미널 안 엶 (가장 흔함)

rc 파일을 고쳐도 현재 셸은 자동으로 다시 읽지 않습니다. 변경을 적용하려면 둘 중 하나:

source ~/.zshrc        # 현재 셸에 즉시 반영
# 또는 터미널 창을 완전히 닫고 새로 열기

T-2. 따옴표 안의 ~ 는 확장되지 않음 (대표 함정)

가장 많이 하는 실수입니다. 큰따옴표 안의 ~는 홈 디렉터리로 치환되지 않아 PATH에 ~/.local/bin이라는 글자 그대로 들어갑니다 → 셸이 못 찾음.

# ✗ 잘못 — 따옴표 안 ~ 는 확장 안 됨
export PATH="~/.local/bin:$PATH"

# ✓ 올바름 — $HOME 사용 (또는 따옴표 밖에 ~)
export PATH="$HOME/.local/bin:$PATH"
확인

이미 잘못 넣었다면 echo $PATH | tr ':' '\n' | grep local로 ~/.local/bin(물결표 그대로)이 보이는지 확인 후 rc 파일에서 수정하세요.

T-3. 셸 불일치 — 설치는 zsh 가정, 실제는 bash (또는 반대)

설치 스크립트가 한쪽 rc에만 export를 추가했는데, 실제 사용 셸이 다른 경우입니다.

echo $SHELL          # /bin/zsh 인지 /bin/bash 인지 확인
echo $0              # 현재 실행 중인 셸 이름

본인 셸의 rc 파일에 직접 추가하세요. zsh → ~/.zshrc, bash → ~/.bashrc. 두 셸을 오가며 쓴다면 양쪽 모두 추가하면 됩니다.

T-4. zsh: .zshrc는 읽는데 PATH가 사라짐 — 로그인 셸과 .zprofile

macOS Terminal/iTerm은 새 창을 보통 로그인 셸로 엽니다. zsh 로딩 순서는 .zprofile → .zshrc입니다. .zprofile(또는 /etc/paths)에서 PATH를 통째로 재설정하면 .zshrc에서 앞에 붙인 항목이 밀려날 수 있습니다.

  • 대부분의 PATH 추가는 ~/.zshrc에 두는 것으로 충분합니다.
  • 로그인 셸에서만 PATH가 빠진다면 ~/.zprofile에도 동일 export를 추가하세요.
  • .zprofile이 PATH를 덮어쓰지 않는지(=export PATH="..."로 통째 재할당하는 줄이 있는지) 확인하세요.
# 로그인 셸에서도 보장하려면
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zprofile
source ~/.zprofile

T-5. bash: macOS에서 .bashrc 가 안 읽힘 — .bash_profile

macOS에서 bash는 로그인 셸일 때 .bashrc가 아니라 .bash_profile(또는 .profile)을 읽습니다. .bashrc에만 넣으면 새 창에서 적용 안 될 수 있습니다.

# .bash_profile 에서 .bashrc 를 불러오게 연결 (한 번만)
echo '[ -f ~/.bashrc ] && . ~/.bashrc' >> ~/.bash_profile

# 그리고 PATH 는 .bashrc 에 추가
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bash_profile

T-6. 중복 export / 순서 문제

같은 export가 여러 줄 쌓였거나, 뒤쪽 줄이 PATH를 덮어쓰면 의도와 달라집니다.

# rc 파일에서 중복 라인 확인
grep -n 'local/bin' ~/.zshrc ~/.zprofile ~/.bashrc ~/.bash_profile 2>/dev/null

# 현재 PATH 항목을 줄 단위로 보기 (중복/순서 점검)
echo $PATH | tr ':' '\n'

중복 줄은 정리하고, ~/.local/bin이 다른 동일 이름 바이너리보다 앞에 오도록 :$PATH 형태(앞에 prepend)로 유지하세요.

T-7. PATH는 맞는데 바이너리 권한 문제

# 실행 권한/존재 확인
ls -la ~/.local/bin/claude

# 디렉터리 쓰기 권한 확인
test -w ~/.local/bin && echo writable || echo "not writable"

# 소유권이 root 등으로 꼬였다면 (sudo 설치 후유증)
sudo chown -R $(whoami) ~/.local
주의

과거 sudo로 설치했다면 소유권이 root로 잡혀 PATH가 맞아도 실행/업데이트가 막힐 수 있습니다. 위 chown으로 사용자 소유로 되돌리세요.

T-8. VS Code·IDE 내장 터미널에서만 못 찾음

IDE 내장 터미널이 로그인 셸 환경을 상속하지 못한 경우입니다. 일반 터미널에선 되는데 IDE에서만 안 되면:

  • IDE를 완전히 종료 후, PATH가 잡힌 터미널에서 IDE를 실행해 환경을 상속시키기
  • 또는 IDE 설정에서 통합 터미널을 "로그인 셸로 실행" 옵션 활성화
  • 그래도 안 되면 IDE 터미널에서 source ~/.zshrc 후 which claude 확인

T-9. 마지막 수단 — 종합 진단

claude doctor   # PATH·권한·Keychain·업데이트 상태 한 번에 점검

그래도 해결이 안 되면 claude --version 동작 여부, 실행한 설치 명령, 전체 에러 출력을 모아 GitHub Issues에서 동일 사례를 검색하거나 새로 등록하세요.

흐름도진단 분기: claude --version 실패 → 바이너리 존재? (없음→재설치 / 있음→PATH 점검) → 따옴표 ~ 함정 → 셸 불일치 → 로그인 셸 rc(.zprofile/.bash_profile) → 권한 → claude doctor
